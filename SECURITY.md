# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it
responsibly.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via [GitHub Security Advisories][github-security-advisories].

You should receive a response within 72 hours. If the vulnerability is confirmed,
a fix will be released as soon as possible.

## Disclosure Policy

When a security bug is reported, the following steps are taken:

1. Confirm the problem and determine the affected versions.
2. Audit code to find any similar problems.
3. Prepare fixes for all supported versions.
4. Release new versions as soon as possible.

[github-security-advisories]: https://github.com/sheeeng/openssf-scorecard-example/security/advisories/new
